How Hybrid Encryption Works
  1. A random AES-256 session key is generated for this message only.
  2. Your message is encrypted with AES-256-GCM (authenticated encryption).
  3. The session key is encrypted with the recipient's RSA public key using OAEP padding.
  4. The output JSON contains encrypted_key, nonce, and ciphertext.
  5. Only the holder of the matching private key can decrypt the session key and read the message.
